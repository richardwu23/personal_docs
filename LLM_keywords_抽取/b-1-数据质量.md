# 1. 如何盘点数据

当前阶段最重要的目标不是先做复杂的 prompt 或 normalization，而是先做 **corpus audit（语料盘点 / 语料审计）**，把现有 patent 维度 doc 的真实数据情况摸清楚。

## 1.1 盘点的核心目标

需要先回答这些现实问题：

1. patent doc 里到底有哪些字段
2. 哪些字段文本质量高
3. 哪些字段最适合做 keyword search
4. 哪些字段更适合做 semantic search
5. ingredients / formulation / process / property / application / dosage form 这些信息通常出现在哪些字段、有哪些常见写法

---

## 1.2 先做字段盘点，不要一开始就做复杂 NLP

### 第一步：字段清单梳理

先列出索引或原始专利文档中已有字段，例如：

* `title`
* `abstract`
* `claims`
* `description`
* `independent_claims`
* `examples` / `embodiments`
* `assignee`
* `ipc` / `cpc`
* `publication_country`
* `language`
* 其他已有结构化字段

### 第二步：字段覆盖率与质量统计

对每个字段做基础画像：

* 非空率
* 平均长度
* P50 / P90 / P99 长度
* 语言分布
* unique ratio / 重复率
* 是否存在大量 boilerplate、模板化噪声、OCR 噪声、机器翻译痕迹

建议产出一张类似这样的表：

| field       | non_null_rate | avg_len | p90_len | 备注         |
| ----------- | ------------: | ------: | ------: | ---------- |
| title       |          0.99 |      88 |     130 | 高价值        |
| abstract    |          0.95 |    1200 |    2500 | 高价值        |
| claims      |          0.90 |    8000 |   25000 | 信息丰富但噪声较高  |
| description |          0.82 |   30000 |  100000 | 很丰富但过长     |
| ipc/cpc     |          0.97 |       - |       - | 更适合 filter |

### 第三步：字段抽样人工阅读

每个关键字段抽样 100～300 条做人工 review，重点看：

* ingredient 是否主要出现在 title / abstract / claims / description
* process 是否主要集中在 description / examples
* dosage form 在 title / abstract 中是否有稳定表达
* 属性词是不是在 claims 中更丰富但更噪
* application 是否有稳定术语

这一步很关键，因为后面 query understanding 和 normalization 都依赖对真实语料的理解。

## 1.3 做术语分布盘点，而不是直接上实体抽取

当前阶段建议先做 **term profiling**，先看“真实语料里怎么写”，而不是直接做复杂 NER。

### 对 ingredients

重点看：

* 候选术语频次
* 大小写变体
* 缩写 / 全称变体
* 连字符变体
* 英文 / 中文 / 化学式混写情况
* 常见共现关系

例如常见 pattern：

* `ZnO`
* `TiO2`
* `zinc oxide`
* `titanium dioxide`
* `PEG-100`
* `polymer`
* `glycol`
* `silicone`
* `surfactant`

### 对 formulation

重点看 formulation 的常见表达框架，例如：

* `A composition comprising ...`
* `A formulation including ...`
* `An emulsion containing ...`
* `A cosmetic composition ...`
* `A sunscreen composition ...`

要统计：

* `composition`
* `formulation`
* `emulsion`
* `dispersion`
* `gel`
* `cream`
* `lotion`
* `suspension`

这些词在各字段中的频率和分布稳定性。

### 对 process

重点盘点工艺动词和工艺短语，例如：

* `mixing`
* `heating`
* `stirring`
* `emulsifying`
* `dissolving`
* `dispersing`
* `neutralizing`
* `homogenizing`
* `coating`
* `curing`

并统计：

* 高频 process verbs
* 高频 n-grams，如 `followed by`, `under agitation`, `at elevated temperature`
* 它们主要在哪些字段出现

### 对属性词

属性词很重要，因为用户 query 往往就是围绕属性提的。

例如：

* `transparent`
* `non-greasy`
* `water-resistant`
* `stable`
* `low whitening`
* `glossy`
* `matte`
* `softness`
* `adhesion`
* `viscosity`
* `spreadability`

这里重点找：

* 高频属性词
* 同义 / 近义写法
* 正向 / 负向表达
* 用户语言与专利语言之间是否存在桥接空间

### 对 application / dosage form

重点看是否有稳定表达，例如：

* application: `sunscreen`, `hair care`, `skin care`, `coating`, `adhesive`, `oral care`
* dosage form: `lotion`, `cream`, `gel`, `spray`, `stick`, `suspension`, `emulsion`

要统计这些词在 `title / abstract / claims` 的出现频率和共现模式。

## 1.4 数据工程上具体怎么做

建议走 **SQL / Spark + Python 双轨**。

### A. SQL / Spark / Trino 做基础画像

适合做：

* 字段覆盖率统计
* 长度分布
* 非空率
* 高频 token / n-gram 统计
* 指定词表命中率
* 字段抽样

建议建立几类基础表：

#### `field_stats`

记录字段级统计：

* `field_name`
* `non_null_cnt`
* `avg_char_len`
* `avg_token_len`
* `p50_len`
* `p90_len`
* `p99_len`

#### `patent_corpus_profile_sample`

用于样本审查：

* `doc_id`
* `title`
* `abstract`
* `claims`
* `description`
* `language`
* `ipc/cpc`

#### `term_freq`

记录候选词统计：

* `field_name`
* `term`
* `tf`
* `df`
* `sample_doc_ids`

### B. Python 做更灵活的文本分析

适合做：

* regex 候选抽取
* n-gram 统计
* collocation / PMI
* 变体归并
* 上下文抽样导出

建议准备一些基础脚本：

* `profile_fields.py`
* `extract_candidate_terms.py`
* `build_ngram_stats.py`
* `sample_contexts.py`

## 1.5 第一阶段建议产出的结果

建议最终明确产出这些表 / 文件：

### `field_profile`

字段质量画像

### `term_candidates_ingredient`

ingredient 候选词频表

### `term_candidates_property`

属性词候选表

### `term_candidates_dosage_form`

剂型词候选表

### `term_context_samples`

每个候选词配若干上下文样本，供人工 review

### `field_recommendation`

最后给出字段使用建议，例如：

* keyword search 优先字段：`title + abstract + independent_claims`
* semantic search 优先字段：`abstract + description_chunk`
* process 信息主要在 `description / examples`
* dosage form 在 `title / abstract` 中较稳定
* 属性词在 `claims` 中更丰富，但噪声更高
