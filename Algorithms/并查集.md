在 LeetCode 里，并查集（Union Find / DSU）本质上就做三件事：

1. 判断两个点是否连通
2. 把两个点所在集合合并
3. 维护“当前有几个连通块 / 是否成环 / 某种关系是否矛盾”

所以题目虽然很多，但真正常用的模板并不多。你可以把它归纳成 **4 类模板**：

---

# 1. 最基础模板：数组版并查集

这是最核心、最常用的模板。适用于：

* 点编号是 `0 ~ n-1`
* 只需要维护连通性
* 题目问：

  * 有几个连通块
  * 两点是否连通
  * 加边后是否成环
  * 去掉某些边后是否还能连通

典型题：

* 547. Number of Provinces
* 684. Redundant Connection
* 1319. Number of Operations to Make Network Connected
* 1971. Find if Path Exists in Graph
* 261. Graph Valid Tree（会员题，思想经典）
* 1579. Remove Max Number of Edges to Keep Graph Fully Traversable

---

## 标准代码模板

```python
class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.size = [1] * n
        self.count = n  # 连通块数量，可选

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]

    def union(self, x: int, y: int) -> bool:
        rx = self.find(x)
        ry = self.find(y)
        if rx == ry:
            return False  # 本来就在一个集合里，合并失败

        # 按 size 合并，小树挂大树
        if self.size[rx] < self.size[ry]:
            rx, ry = ry, rx

        self.parent[ry] = rx
        self.size[rx] += self.size[ry]
        self.count -= 1
        return True

    def connected(self, x: int, y: int) -> bool:
        return self.find(x) == self.find(y)
```

---

## 这个模板的三种最常见用法

### 用法 1：判断是否成环

典型题：684 Redundant Connection

```python
def findRedundantConnection(edges):
    n = len(edges)
    uf = UnionFind(n + 1)  # 因为题目节点从 1 开始

    for a, b in edges:
        if not uf.union(a, b):
            return [a, b]   # 这条边让图成环
```

核心思想：

* 如果一条边两端本来就连通，再加这条边就成环

---

### 用法 2：统计连通块数量

典型题：547 Number of Provinces

```python
def findCircleNum(isConnected):
    n = len(isConnected)
    uf = UnionFind(n)

    for i in range(n):
        for j in range(i + 1, n):
            if isConnected[i][j] == 1:
                uf.union(i, j)

    return uf.count
```

核心思想：

* 初始每个点一个集合
* 每成功 union 一次，连通块数减 1

---

### 用法 3：判断最终是否完全连通

典型题：1971 / 1319

```python
def validPath(n, edges, source, destination):
    uf = UnionFind(n)
    for a, b in edges:
        uf.union(a, b)
    return uf.connected(source, destination)
```

# 2. 网格题模板：二维坐标映射到一维

这一类其实还是标准并查集，只是把 `(r, c)` 映射成一个整数编号。

典型题：

* 200. Number of Islands
* 305. Number of Islands II
* 695. Max Area of Island（更常见 DFS，但也能用并查集）
* 827. Making A Large Island（常见 DFS，也可并查集）

---

## 坐标映射公式

对于 `rows x cols` 的网格：

```python
idx = r * cols + c
```

---

## 模板代码（以 200 Number of Islands 为例）

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.size[rx] < self.size[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        self.size[rx] += self.size[ry]
        return True


def numIslands(grid):
    if not grid or not grid[0]:
        return 0

    rows, cols = len(grid), len(grid[0])
    uf = UnionFind(rows * cols)
    water = 0

    def index(r, c):
        return r * cols + c

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "0":
                water += 1
                continue

            for dr, dc in [(1, 0), (0, 1)]:  # 只看右、下，避免重复
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == "1":
                    uf.union(index(r, c), index(nr, nc))

    roots = set()
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                roots.add(uf.find(index(r, c)))

    return len(roots)
```

---

## 网格题更实用的写法

很多时候会加一个 `count`，并在遇到陆地时初始化统计，这样更方便：

```python
class UnionFind:
    def __init__(self, grid):
        rows, cols = len(grid), len(grid[0])
        n = rows * cols
        self.parent = list(range(n))
        self.size = [1] * n
        self.count = 0
        self.rows = rows
        self.cols = cols

        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == "1":
                    self.count += 1

    def idx(self, r, c):
        return r * self.cols + c

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.size[rx] < self.size[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        self.size[rx] += self.size[ry]
        self.count -= 1
        return True
```

这样最后直接返回 `uf.count`。

# 3. 哈希映射版并查集：元素不是数字编号

这一类题的特点是：

* 节点不是 `0~n-1`
* 可能是字符串、邮箱、变量名
* 需要先“离散化”或者直接用字典做并查集

典型题：

* 721. Accounts Merge
* 1202. Smallest String With Swaps
* 839. Similar String Groups
* 947. Most Stones Removed with Same Row or Column
* 990. Satisfiability of Equality Equations（字符映射也可直接数组）

---

## 字典版并查集模板

```python
class UnionFind:
    def __init__(self):
        self.parent = {}
        self.size = {}

    def add(self, x):
        if x not in self.parent:
            self.parent[x] = x
            self.size[x] = 1

    def find(self, x):
        self.add(x)
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx = self.find(x)
        ry = self.find(y)
        if rx == ry:
            return False

        if self.size[rx] < self.size[ry]:
            rx, ry = ry, rx

        self.parent[ry] = rx
        self.size[rx] += self.size[ry]
        return True
```

---

## 典型题：721 Accounts Merge

```python
from collections import defaultdict

def accountsMerge(accounts):
    uf = UnionFind()
    email_to_name = {}

    for account in accounts:
        name = account[0]
        first_email = account[1]

        for email in account[1:]:
            uf.union(first_email, email)
            email_to_name[email] = name

    groups = defaultdict(list)
    for email in email_to_name:
        root = uf.find(email)
        groups[root].append(email)

    res = []
    for root, emails in groups.items():
        res.append([email_to_name[root]] + sorted(emails))

    return res
```

这类题的核心套路是：

1. 先把“有关系”的字符串 union 起来
2. 再按 root 分组

# 4. 带权并查集 / 关系并查集：维护额外信息

这是进阶模板。普通并查集只能维护“是否同组”，但有些题还需要维护：

* 两点之间的比值
* 两点之间的相对关系
* 奇偶性 / 敌我关系 / 距离差

典型题：

* 399. Evaluate Division
* 某些“食物链”“敌人朋友”“奇偶约束”类题
* 部分竞赛题、面试进阶题里常见

LeetCode 日常高频里，你先把基础并查集掌握好，再看这一类。

---

## 带权并查集模板（以比值为例）

设：

* `weight[x]` 表示 `x / parent[x]` 的值

那么路径压缩时要同步更新权重。

```python
class UnionFind:
    def __init__(self):
        self.parent = {}
        self.weight = {}  # x / parent[x]

    def add(self, x):
        if x not in self.parent:
            self.parent[x] = x
            self.weight[x] = 1.0

    def find(self, x):
        self.add(x)
        if self.parent[x] != x:
            origin = self.parent[x]
            self.parent[x] = self.find(origin)
            self.weight[x] *= self.weight[origin]
        return self.parent[x]

    def union(self, x, y, value):
        """
        已知 x / y = value
        """
        self.add(x)
        self.add(y)

        rx = self.find(x)
        ry = self.find(y)

        if rx == ry:
            return

        # 令 rx 挂到 ry
        # 已知:
        # x / y = value
        # x / rx = weight[x]
        # y / ry = weight[y]
        # 需要求 rx / ry
        self.parent[rx] = ry
        self.weight[rx] = value * self.weight[y] / self.weight[x]

    def is_connected(self, x, y):
        return x in self.parent and y in self.parent and self.find(x) == self.find(y)

    def ratio(self, x, y):
        if not self.is_connected(x, y):
            return -1.0
        return self.weight[x] / self.weight[y]
```

# 并查集题目，按“题型”怎么记

这个比死记题号更有用。

---

## A. 连通块计数

题目特征：

* “有多少个省份 / 岛屿 / 连通分量”

代表题：

* 547 Number of Provinces
* 200 Number of Islands
* 1319 Number of Operations to Make Network Connected

套路：

* `count = n`
* 每次 `union` 成功就 `count -= 1`

---

## B. 成环检测 / 冗余边

题目特征：

* “哪条边是多余的”
* “这张图是不是树”
* “加上这条边会不会形成环”

代表题：

* 684 Redundant Connection
* 261 Graph Valid Tree

套路：

* 如果 `find(a) == find(b)`，说明已连通，再连就成环

---

## C. 动态连通性

题目特征：

* 多次加边
* 问某时刻两点是否连通

代表题：

* 1971 Find if Path Exists in Graph
* 305 Number of Islands II

套路：

* 每次操作 `union`
* 查询时 `connected(x, y)`

---

## D. 分组归类

题目特征：

* 某些元素因为共享属性而属于同一组
* 最后要把组里的元素收集出来

代表题：

* 721 Accounts Merge
* 1202 Smallest String With Swaps
* 839 Similar String Groups

套路：

* union 相关元素
* 最后 `root -> list` 分组

---

## E. 关系约束 / 方程约束

题目特征：

* `a == b`
* `a / b = k`
* 某种相对关系是否矛盾

代表题：

* 990 Satisfiability of Equality Equations
* 399 Evaluate Division

套路：

* 简单相等关系：普通并查集
* 带比值/偏移：带权并查集

# 面试里最值得背熟的两个模板

如果你只想先掌握最有用的，建议先背下面两个。

---

## 模板 1：数组版 DSU

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n
        self.count = n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]  # 路径压缩（路径减半）
            x = self.parent[x]
        return x

    def union(self, x, y):
        rx = self.find(x)
        ry = self.find(y)
        if rx == ry:
            return False
        if self.size[rx] < self.size[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        self.size[rx] += self.size[ry]
        self.count -= 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)
```

这个是迭代版 `find`，面试里也很好用。

# 并查集和 DFS/BFS 的区别

这个面试里也常问。

### 什么时候更适合并查集

* 图是**动态加边**的
* 需要频繁判断“两个点现在是否连通”
* 需要快速合并集合
* 需要判断成环
* 需要做 Kruskal 最小生成树

### 什么时候 DFS/BFS 更自然

* 一次性遍历整张图
* 需要路径细节
* 需要遍历某个连通块里的所有节点
* 岛屿题中只求个数时 DFS 代码通常更直观

# 你可以直接套用的思维模板

以后看到题，先问自己：

### 1. 题目是不是在问“连通性”？

如果是，先想并查集。

### 2. 有没有“合并关系”？

比如：

* 同邮箱
* 同一行/列
* 同一个字符串组
* 相等关系
* 图中的边

如果有，适合并查集。

### 3. 有没有“加一条边后是否成环”？

这是并查集经典信号。

### 4. 元素是不是字符串或对象？

如果是，用字典版并查集。

### 5. 是二维网格吗？

如果是，把 `(r, c)` 映射成一维编号。
